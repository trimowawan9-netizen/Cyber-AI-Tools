3.3 XSS Cross Site Scripting

⚠️ ETIKA: XSS cuma boleh diuji di DVWA, PortSwigger Lab, atau web milik sendiri. Nyuntik JS ke web asli = nyolong data orang = melanggar hukum.

1. Apa itu XSS?

XSS = attacker nyelipin kode JavaScript ke web korban. Ibarat: Kamu nulis komentar "Halo", tapi webnya nurut kalau kamu tulis "Halo + nyalain kamera korban".

2. 3 Jenis XSS Paling Umum

1. Reflected XSS
Script cuma jalan 1x pas korban klik link jahat. Contoh: link search di web
2. Stored XSS
Script disimpan di database. Jalan tiap ada orang buka halaman itu. Paling bahaya.
3. DOM XSS
JS di browser korban yang gak aman. Script gak ke server, tapi browsernya yang ketipu.

3. Dampak Kalau Kena XSS

1. Cookie Theft: Maling session login korban
2. Keylogger: Catat semua yang diketik korban
3. Deface: Ganti tampilan web di browser korban
4. Redirect: Lempar korban ke web phishing

4. Cara Defense XSS = Jadi Developer Aman

1. Output Encoding / Escape
Semua input user diubah jadi teks biasa. Contoh: < jadi &lt;
Bahasa: PHP pakai htmlspecialchars(), JS pakai textContent

2. Content Security Policy CSP
Bilang ke browser: "Cuma boleh jalanin JS dari domain gua sendiri"

3. Validasi Input
Komentar cuma boleh huruf/angka. Tag HTML <script> langsung dibuang.

4. HttpOnly Cookie
Cookie login gak bisa diakses JS. Jadi walau kena XSS, cookie aman.

5. Lab Legal Buat Latihan XSS

1. DVWA → Set level low → liat XSS reflected
2. PortSwigger Web Security Academy → Gratis + sertifikat
3. XSS Game Google → Game edukasi XSS
🔥 Latihan Aman 2 Menit:
1. Buka web kamu sendiri → Inspect → Console
2. Ketik: document.body.innerHTML = '<h1>Lu Kena XSS</h1>'
3. Itu DOM XSS versi lokal. Cuma ngerubah di browser kamu doang, aman.
4. Sekarang paham kan kenapa dev harus filter input?

XSS Hunter = pemburu bayaran tertinggi di Bug Bounty. 1 XSS Stored bisa $1000+. Tapi kuncinya: jago defense dulu.
