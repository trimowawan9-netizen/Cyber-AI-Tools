3.2 SQL Injection - Konsep & Pencegahan

⚠️ ETIKA & HUKUM: Pengujian SQLi hanya boleh di website milik sendiri atau dengan izin tertulis. Menyerang web orang = pelanggaran UU ITE. Materi ini 100% untuk edukasi & defense.

1. Apa itu SQL Injection?

SQLi terjadi ketika website langsung masukin input user ke query database tanpa filter. Ibarat: Kasir nanya "Nama?", kamu jawab "Nama + ambil semua uang di laci" → kasir nurut.

2. Cara Kerja Basic

Query normal: SELECT * FROM users WHERE username='input_user'
Kalau input gak disaring, attacker bisa "ngubah" arti query-nya.
Hasil: Data bocor, login kebobolan, database kehapus.

3. Cara Mencegah SQLi = Jadi Developer Aman

1. Prepared Statement / Parameterized Query
Pisahin kode SQL sama data user. Database gak akan anggap input sebagai kode.

2. Input Validation
Cek input: username cuma boleh huruf+angka, password min 8 karakter.

3. Least Privilege
Akun database web cuma boleh SELECT/INSERT. Jangan kasih hak DROP TABLE.

4. WAF + Error Handling
Sembunyiin error database asli. Tampilkan "Login gagal" aja.

4. Tools Legal untuk Test Web Sendiri

1. DVWA - Damn Vulnerable Web App: web latihan SQLi legal
2. SQLMap - Auto test SQLi, tapi cuma di lab kamu
3. Burp Suite Community - Scanner buat web kamu sendiri
🔥 Latihan Aman untuk Pemula:
1. Install XAMPP + DVWA di laptop kamu
2. Set DVWA ke level "low" → coba pahami gimana SQLi terjadi
3. Naikin ke "medium/high" → liat gimana Prepared Statement ngeblok serangan
4. Goal: Bisa jelasin ke temen "kenapa web harus pake prepared statement"

Jadi hacker white hat = jago defense. Bisa bikin web anti bobol = skill paling laku di Bug Bounty & kerjaan.
