2.3 DNS & HTTP

Cara domain jadi website, Request & Response. Dari ketik sampe muncul web

1. DNS = Buku Telepon Internet

Komputer gak ngerti "google.com". Dia cuma ngerti IP. DNS tugasnya nerjemahin:

google.com → DNS Server → 142.250.66.78
Ibarat: "Pak RT" → "Buku Telepon" → "Alamat lengkap rumah"
nslookup google.com → Liat IP google
ping google.com → Cek koneksi + IP
8.8.8.8 = DNS Google, 1.1.1.1 = DNS Cloudflare

2. HTTP Request & Response = Tanya Jawab

Browser sama server ngobrol pake HTTP. Formatnya gini:

REQUEST:
GET /login HTTP/1.1
Host: target.com
User-Agent: Chrome

RESPONSE:
HTTP/1.1 200 OK
Content-Type: text/html
<html>...isi web...</html>

3. Status Code = Bahasa Server

200 OK = Sukses, web muncul
301/302 = Pindah alamat, redirect
403 Forbidden = Dilarang masuk, akses ditolak
404 Not Found = Halaman gak ada
500 Error = Servernya error sendiri
🔥 Latihan 1 Menit - Jadi Hacker:
1. Buka Chrome → F12 → tab Network → refresh web
2. Klik file "document" → liat tab Headers
3. Itu isi Request & Response beneran! Cari "Status Code: 200"
4. CMD ketik: nslookup facebook.com → liat IP-nya
