3.4 CSRF Cross Site Request Forgery

⚠️ ETIKA: CSRF cuma boleh diuji di DVWA, OWASP WebGoat, atau web lab sendiri. Bikin form jahat ke web asli = ilegal, sama aja nyolong duit orang.

1. Apa itu CSRF?

Analogi: Kamu login ke ATM, terus ada orang nyelipin kertas "Transfer 10jt ke saya" ke tombol ATM. Kamu kira mau pencet "Cek Saldo", tapi yang kepencet kertas itu.

CSRF = attacker nyuruh browser korban ngirim request ke web lain tanpa korban sadar. Syaratnya: korban harus lagi login di web target.

2. Target CSRF Paling Bahaya

1. Ganti Password/Email: Akun korban diambil alih
2. Transfer Dana: E-wallet, bank, crypto
3. Post Komentar: Spam dari akun korban
4. Ganti Setting: Matikan 2FA korban

3. Cara Defense CSRF = Wajib Pake Ini

1. CSRF Token
Setiap form kasih kode rahasia random. Server cek: "kodenya bener gak?"
Kalau request dari web jahat = gak punya token = ditolak.

2. SameSite Cookie
Set cookie: SameSite=Strict. Browser gak akan bawa cookie login kalau request dari web lain.

3. Cek Referer/Origin Header
Server nanya: "Request ini dari web gua sendiri gak?" Kalau dari web lain = blokir.

4. Konfirmasi User
Aksi penting kayak transfer/ganti password wajib minta password lagi.

4. Lab Legal Buat Paham CSRF

1. DVWA → Menu CSRF → level low ke high
2. OWASP WebGoat → Modul CSRF lengkap
3. PortSwigger CSRF Lab → Gratis, ada hintnya
🔥 Latihan Paham Konsep:
1. Buka 2 tab: Tab 1 login Facebook, Tab 2 buka web random
2. Pikir: Kenapa FB gak bisa diganti fotonya dari Tab 2?
3. Jawaban: Karena FB pake CSRF Token + SameSite Cookie
4. Itu defense yang bikin hacker CSRF frustasi

CSRF gampang dicegah tapi sering dilupain dev pemula. Jadi kalau kamu paham ini = skill kamu udah di atas rata-rata developer Indo.
